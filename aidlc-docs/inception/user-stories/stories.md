# User Stories - 테이블오더 서비스

## Story Format
- 세분화 수준: Task 수준 (세부 동작별)
- Acceptance Criteria: 상세 (Given-When-Then + 엣지 케이스 + 에러 시나리오)
- 구성: Persona-Based + User Journey 순서

---

# 고객 (Customer) Stories

## US-C01: 테이블 태블릿 초기 화면 진입
**As a** 고객, **I want to** 테이블 태블릿에서 별도 로그인 없이 메뉴 화면을 바로 볼 수 있기를 **so that** 즉시 주문을 시작할 수 있다.

**Acceptance Criteria**:
- Given 태블릿에 자동 로그인 정보가 저장되어 있을 때, When 브라우저를 열면, Then 메뉴 화면이 기본 화면으로 표시된다
- Given 자동 로그인 정보가 없을 때, When 브라우저를 열면, Then 초기 설정 화면(매장ID/테이블번호/비밀번호 입력)이 표시된다
- Edge: 저장된 로그인 정보가 만료된 경우, 초기 설정 화면으로 리다이렉트
- Error: 서버 연결 실패 시 "서버에 연결할 수 없습니다" 에러 메시지 표시

---

## US-C02: 테이블 태블릿 초기 설정
**As a** 매장 관리자(태블릿 설정 시), **I want to** 매장 식별자, 테이블 번호, 테이블 비밀번호를 입력하여 태블릿을 설정할 수 있기를 **so that** 고객이 이후 자동 로그인으로 사용할 수 있다.

**Acceptance Criteria**:
- Given 초기 설정 화면에서, When 매장 식별자/테이블 번호/비밀번호를 입력하고 설정 버튼을 누르면, Then 서버 인증 후 로그인 정보가 로컬에 저장된다
- Given 설정 성공 시, When 저장이 완료되면, Then 메뉴 화면으로 자동 이동한다
- Edge: 이미 설정된 태블릿에서 재설정 시 기존 정보 덮어쓰기
- Error: 잘못된 매장 식별자/비밀번호 입력 시 "인증 정보가 올바르지 않습니다" 에러 표시

---

## US-C03: 카테고리별 메뉴 목록 조회
**As a** 고객, **I want to** 카테고리별로 분류된 메뉴 목록을 볼 수 있기를 **so that** 원하는 종류의 메뉴를 쉽게 찾을 수 있다.

**Acceptance Criteria**:
- Given 메뉴 화면에서, When 페이지가 로드되면, Then 카테고리 탭과 해당 카테고리의 메뉴가 카드 형태로 표시된다
- Given 메뉴 카드에, When 표시될 때, Then 메뉴명, 가격, 이미지가 포함된다
- Given 카테고리 탭에서, When 다른 카테고리를 탭하면, Then 해당 카테고리의 메뉴로 빠르게 이동한다
- Edge: 카테고리에 메뉴가 없는 경우 "등록된 메뉴가 없습니다" 표시
- Error: 메뉴 데이터 로드 실패 시 재시도 버튼 표시

---

## US-C04: 메뉴 상세 정보 조회
**As a** 고객, **I want to** 메뉴의 상세 정보(설명, 이미지)를 볼 수 있기를 **so that** 주문 전에 메뉴에 대해 충분히 알 수 있다.

**Acceptance Criteria**:
- Given 메뉴 카드를 탭했을 때, When 상세 화면이 열리면, Then 메뉴명, 가격, 설명, 이미지가 표시된다
- Given 상세 화면에서, When 장바구니 추가 버튼을 누르면, Then 커스텀 옵션 선택 화면이 표시된다 (옵션이 있는 메뉴의 경우)
- Given 옵션이 없는 메뉴에서, When 장바구니 추가 버튼을 누르면, Then 바로 장바구니에 추가된다
- Edge: 이미지가 없는 메뉴는 기본 플레이스홀더 이미지 표시
- Edge: 설명이 없는 메뉴는 설명 영역 미표시

---

## US-C04-1: 커스텀 옵션 선택
**As a** 고객, **I want to** 커피 주문 시 사이즈, 온도, 샷 추가, 시럽, 우유 종류 등 커스텀 옵션을 선택할 수 있기를 **so that** 나만의 맞춤 커피를 주문할 수 있다.

**Acceptance Criteria**:
- Given 커스텀 옵션 선택 화면에서, When 옵션 그룹이 표시되면, Then 각 그룹별 옵션 목록과 추가 금액이 표시된다
- Given 필수 옵션 그룹에서, When 옵션을 선택하지 않으면, Then 장바구니 추가 버튼이 비활성화된다
- Given 단일 선택 옵션 그룹에서, When 옵션을 선택하면, Then 이전 선택이 해제되고 새 옵션이 선택된다
- Given 다중 선택 옵션 그룹에서, When 옵션을 선택하면, Then 기존 선택에 추가된다
- Given 옵션 선택 시, When 추가 금액이 있으면, Then 총 가격이 실시간으로 업데이트된다
- Given 모든 필수 옵션 선택 후, When 장바구니 추가 버튼을 누르면, Then 선택한 옵션과 함께 장바구니에 추가된다
- Edge: 같은 메뉴라도 옵션이 다르면 별도 항목으로 장바구니에 추가

---

## US-C05: 장바구니에 메뉴 추가
**As a** 고객, **I want to** 원하는 메뉴를 장바구니에 추가할 수 있기를 **so that** 여러 메뉴를 모아서 한 번에 주문할 수 있다.

**Acceptance Criteria**:
- Given 커스텀 옵션 선택 완료 후, When 장바구니 추가 버튼을 누르면, Then 해당 메뉴가 선택한 옵션과 함께 장바구니에 추가된다
- Given 옵션이 없는 메뉴에서, When 장바구니 추가 버튼을 누르면, Then 수량 1로 장바구니에 추가된다
- Given 장바구니에 동일 메뉴+동일 옵션 조합이 있을 때, When 다시 추가하면, Then 수량이 1 증가한다
- Given 장바구니에 동일 메뉴+다른 옵션 조합일 때, When 추가하면, Then 별도 항목으로 추가된다
- Given 장바구니에 메뉴가 추가되면, When 화면에 표시될 때, Then 장바구니 아이콘에 총 수량이 표시된다
- Edge: 장바구니 데이터는 로컬 저장되어 페이지 새로고침 시에도 유지
- Edge: 장바구니 항목에 선택한 커스텀 옵션이 표시된다

---

## US-C06: 장바구니 수량 조절
**As a** 고객, **I want to** 장바구니에 담긴 메뉴의 수량을 조절할 수 있기를 **so that** 원하는 만큼 주문할 수 있다.

**Acceptance Criteria**:
- Given 장바구니 화면에서, When 수량 증가(+) 버튼을 누르면, Then 해당 메뉴 수량이 1 증가하고 총 금액이 재계산된다
- Given 장바구니 화면에서, When 수량 감소(-) 버튼을 누르면, Then 해당 메뉴 수량이 1 감소하고 총 금액이 재계산된다
- Given 수량이 1인 메뉴에서, When 수량 감소 버튼을 누르면, Then 해당 메뉴가 장바구니에서 삭제된다
- Edge: 수량 변경 시 총 금액이 실시간으로 업데이트

---

## US-C07: 장바구니 메뉴 삭제
**As a** 고객, **I want to** 장바구니에서 특정 메뉴를 삭제할 수 있기를 **so that** 원하지 않는 메뉴를 제거할 수 있다.

**Acceptance Criteria**:
- Given 장바구니 화면에서, When 메뉴 삭제 버튼을 누르면, Then 해당 메뉴가 장바구니에서 즉시 제거된다
- Given 메뉴 삭제 후, When 총 금액이 표시될 때, Then 삭제된 메뉴 금액이 차감된 총 금액이 표시된다
- Edge: 마지막 메뉴 삭제 시 장바구니 비어있음 상태 표시

---

## US-C08: 장바구니 비우기
**As a** 고객, **I want to** 장바구니를 한 번에 비울 수 있기를 **so that** 처음부터 다시 메뉴를 선택할 수 있다.

**Acceptance Criteria**:
- Given 장바구니에 메뉴가 있을 때, When 장바구니 비우기 버튼을 누르면, Then 모든 메뉴가 삭제되고 총 금액이 0원이 된다
- Edge: 빈 장바구니에서 비우기 버튼은 비활성화

---

## US-C09: 주문 내역 확인 및 주문 확정
**As a** 고객, **I want to** 장바구니의 주문 내역을 최종 확인하고 주문을 확정할 수 있기를 **so that** 실수 없이 정확한 주문을 할 수 있다.

**Acceptance Criteria**:
- Given 장바구니 화면에서, When 주문하기 버튼을 누르면, Then 주문 내역(메뉴명, 커스텀 옵션, 수량, 단가, 총 금액)이 확인 화면에 표시된다
- Given 주문 확인 화면에서, When 주문 확정 버튼을 누르면, Then 서버에 주문이 전송된다
- Given 주문 성공 시, When 응답을 받으면, Then 주문 번호가 표시되고 5초 후 메뉴 화면으로 자동 리다이렉트된다
- Given 주문 성공 시, When 장바구니가 처리될 때, Then 장바구니가 자동으로 비워진다
- Error: 주문 실패 시 에러 메시지 표시 및 장바구니 유지
- Edge: 빈 장바구니에서 주문하기 버튼은 비활성화
- Edge: 해당 테이블의 첫 주문인 경우, 주문 성공 시 새로운 테이블 세션이 자동 생성된다

---

## US-C10: 주문 내역 목록 조회
**As a** 고객, **I want to** 현재 테이블 세션의 주문 내역을 볼 수 있기를 **so that** 이전에 주문한 내용을 확인할 수 있다.

**Acceptance Criteria**:
- Given 주문 내역 화면에서, When 페이지가 로드되면, Then 현재 세션의 주문 목록이 시간 순으로 표시된다
- Given 각 주문 항목에, When 표시될 때, Then 주문 번호, 주문 시각, 메뉴 및 수량, 커스텀 옵션, 금액, 상태가 포함된다
- Given 주문 상태가 변경되면, When SSE 이벤트를 수신하면, Then 해당 주문의 상태가 실시간으로 업데이트된다
- Edge: 현재 세션 주문만 표시 (이전 세션 주문 제외)
- Edge: 주문이 없는 경우 "주문 내역이 없습니다" 표시

# 매장 관리자 (Store Admin) Stories

## US-A01: 관리자 역할 선택 진입
**As a** 매장 관리자, **I want to** 역할 선택 화면에서 매장 관리자를 선택하고 매장을 선택하여 바로 진입할 수 있기를 **so that** 별도 로그인 없이 빠르게 관리 화면에 접근할 수 있다.

**Acceptance Criteria**:
- Given 관리자 페이지 접속 시, When 역할 선택 화면이 표시되면, Then 슈퍼 관리자/매장 관리자 선택 옵션이 표시된다
- Given 매장 관리자를 선택했을 때, When 매장 목록이 표시되면, Then 등록된 매장 목록에서 매장을 선택할 수 있다
- Given 매장을 선택하면, When 진입이 완료되면, Then 해당 매장의 주문 모니터링 대시보드가 표시된다
- Edge: 등록된 매장이 없는 경우 "등록된 매장이 없습니다" 표시

---

## US-A02: 실시간 주문 대시보드 조회
**As a** 매장 관리자, **I want to** 테이블별 주문 현황을 그리드 형태의 대시보드로 볼 수 있기를 **so that** 매장 전체 주문 상황을 한눈에 파악할 수 있다.

**Acceptance Criteria**:
- Given 주문 모니터링 화면에서, When 페이지가 로드되면, Then 테이블별 카드가 그리드 레이아웃으로 표시된다
- Given 각 테이블 카드에, When 표시될 때, Then 테이블 번호, 총 주문액, 최신 주문 미리보기가 포함된다
- Given SSE 연결이 활성화되면, When 신규 주문이 들어오면, Then 2초 이내에 해당 테이블 카드가 업데이트된다
- Given 신규 주문이 들어오면, When 카드가 업데이트될 때, Then 시각적 강조(색상 변경, 애니메이션)가 적용된다
- Edge: 주문이 없는 테이블은 빈 상태로 표시

---

## US-A03: 주문 상세 보기
**As a** 매장 관리자, **I want to** 테이블 카드를 클릭하여 전체 주문 목록을 상세히 볼 수 있기를 **so that** 해당 테이블의 정확한 주문 내용을 확인할 수 있다.

**Acceptance Criteria**:
- Given 대시보드에서, When 테이블 카드를 클릭하면, Then 해당 테이블의 전체 주문 목록(주문 번호, 시각, 메뉴 및 수량, 커스텀 옵션, 금액)이 표시된다
- Given 상세 보기에서, When 닫기 버튼을 누르면, Then 대시보드로 복귀한다
- Edge: 주문이 없는 테이블 클릭 시 "주문 내역이 없습니다" 표시

---

## US-A04: 주문 상태 변경
**As a** 매장 관리자, **I want to** 주문 상태를 대기중/준비중/완료로 변경할 수 있기를 **so that** 주문 처리 진행 상황을 관리할 수 있다.

**Acceptance Criteria**:
- Given 주문 상세 화면에서, When 상태 변경 버튼을 누르면, Then 대기중→준비중→완료 순서로 상태가 변경된다
- Given 상태가 변경되면, When SSE 이벤트가 발생하면, Then 고객 화면에서도 실시간으로 상태가 업데이트된다
- Given 상태가 변경되면, When 대시보드에 반영될 때, Then 해당 테이블 카드의 주문 상태도 업데이트된다
- Error: 상태 변경 실패 시 에러 메시지 표시 및 이전 상태 유지

---

## US-A05: 테이블별 주문 필터링
**As a** 매장 관리자, **I want to** 특정 테이블의 주문만 필터링하여 볼 수 있기를 **so that** 원하는 테이블의 주문을 빠르게 찾을 수 있다.

**Acceptance Criteria**:
- Given 대시보드에서, When 테이블 필터를 선택하면, Then 선택한 테이블의 카드만 표시된다
- Given 필터가 적용된 상태에서, When 필터를 해제하면, Then 전체 테이블이 다시 표시된다

---

## US-A06: 주문 삭제
**As a** 매장 관리자, **I want to** 특정 주문을 삭제할 수 있기를 **so that** 잘못된 주문을 정정할 수 있다.

**Acceptance Criteria**:
- Given 주문 상세 화면에서, When 주문 삭제 버튼을 누르면, Then 확인 팝업이 표시된다
- Given 확인 팝업에서, When 확인을 누르면, Then 주문이 즉시 삭제되고 테이블 총 주문액이 재계산된다
- Given 확인 팝업에서, When 취소를 누르면, Then 삭제가 취소되고 이전 화면으로 복귀한다
- Given 삭제 성공 시, When 피드백이 표시될 때, Then 성공 메시지가 표시된다
- Error: 삭제 실패 시 에러 메시지 표시

## US-A07: 테이블 이용 완료 처리
**As a** 매장 관리자, **I want to** 테이블 이용 완료를 처리할 수 있기를 **so that** 새 고객이 이전 주문 없이 깨끗한 상태에서 시작할 수 있다.

**Acceptance Criteria**:
- Given 테이블 카드에서, When 이용 완료 버튼을 누르면, Then 확인 팝업이 표시된다
- Given 확인 팝업에서, When 확인을 누르면, Then 해당 세션의 주문 내역이 과거 이력으로 이동된다
- Given 이용 완료 처리 후, When 테이블 카드가 업데이트되면, Then 현재 주문 목록과 총 주문액이 0으로 리셋된다
- Given 이용 완료 처리 후, When 고객 태블릿에서 주문 내역을 조회하면, Then 이전 세션 주문은 표시되지 않는다
- Error: 이용 완료 처리 실패 시 에러 메시지 표시 및 상태 유지

## US-A08: 과거 주문 내역 조회
**As a** 매장 관리자, **I want to** 테이블의 과거 주문 내역을 조회할 수 있기를 **so that** 이전 고객의 주문 이력을 확인할 수 있다.

**Acceptance Criteria**:
- Given 테이블 카드에서, When 과거 내역 버튼을 누르면, Then 해당 테이블의 과거 주문 목록이 시간 역순으로 표시된다
- Given 과거 주문 목록에, When 표시될 때, Then 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 포함된다
- Given 과거 내역 화면에서, When 날짜 필터를 적용하면, Then 해당 날짜의 주문만 표시된다
- Given 과거 내역 화면에서, When 닫기 버튼을 누르면, Then 대시보드로 복귀한다
- Edge: 과거 내역이 없는 경우 "과거 주문 내역이 없습니다" 표시

- Given 입력 폼에서, When 테이블 번호와 비밀번호를 입력하고 저장하면, Then 테이블이 생성되고 16시간 세션이 생성된다
- Error: 중복 테이블 번호 입력 시 "이미 존재하는 테이블 번호입니다" 에러 표시

# 슈퍼 관리자 (Super Admin) Stories

## US-S01: 슈퍼 관리자 역할 선택 진입
**As a** 슈퍼 관리자, **I want to** 역할 선택 화면에서 슈퍼 관리자를 선택하여 바로 진입할 수 있기를 **so that** 시스템 전체 관리 화면에 접근할 수 있다.

**Acceptance Criteria**:
- Given 관리자 페이지 접속 시, When 슈퍼 관리자를 선택하면, Then 슈퍼 관리자 대시보드가 표시된다
- Given 슈퍼 관리자 대시보드에, When 표시될 때, Then 매장 관리, 계정 관리, 메뉴 관리, 카테고리 관리 메뉴가 표시된다

---

## US-S02: 매장 등록
**As a** 슈퍼 관리자, **I want to** 새로운 매장을 등록할 수 있기를 **so that** 시스템에 매장을 추가할 수 있다.

**Acceptance Criteria**:
- Given 매장 관리 화면에서, When 매장 추가 버튼을 누르면, Then 매장 등록 폼(매장명, 매장 식별자 등)이 표시된다
- Given 등록 폼에서, When 필수 정보를 입력하고 저장하면, Then 매장이 등록되고 목록에 표시된다
- Error: 중복 매장 식별자 입력 시 에러 표시
- Error: 필수 필드 미입력 시 검증 에러 표시

---

## US-S03: 매장 목록 조회
**As a** 슈퍼 관리자, **I want to** 등록된 전체 매장 목록을 조회할 수 있기를 **so that** 관리 중인 매장 현황을 파악할 수 있다.

**Acceptance Criteria**:
- Given 매장 관리 화면에서, When 페이지가 로드되면, Then 전체 매장 목록이 표시된다
- Given 매장 목록에, When 표시될 때, Then 매장명, 매장 식별자, 상태 정보가 포함된다
- Edge: 매장이 없는 경우 "등록된 매장이 없습니다" 표시

## US-S04: 매장 정보 수정

**Acceptance Criteria**:
- Given 매장 목록에서, When 매장 수정 버튼을 누르면, Then 기존 정보가 채워진 수정 폼이 표시된다
- Given 수정 폼에서, When 정보를 변경하고 저장하면, Then 매장 정보가 업데이트된다
- Error: 필수 필드 검증 동일 적용

- Given 확인 팝업에서, When 확인을 누르면, Then 매장과 관련 데이터가 삭제된다
- Edge: 활성 주문이 있는 매장 삭제 시 경고 메시지 표시
- Error: 삭제 실패 시 에러 메시지 표시

- Given 생성 폼에서, When 필수 정보를 입력하고 저장하면, Then 계정이 생성되고 목록에 표시된다

**Acceptance Criteria**:
- Given 카테고리 관리 화면에서, When 페이지가 로드되면, Then 기본 카테고리(커피HOT, 커피ICE, 논커피 음료, 스무디/프라페, 티/에이드, 디저트) 목록이 표시된다
- Given 카테고리 관리 화면에서, When 카테고리 추가 버튼을 누르면, Then 카테고리명 입력 폼이 표시된다
- Given 카테고리를 추가/수정/삭제하면, When 저장되면, Then 전체 매장의 고객 화면에 반영된다
- Given 카테고리 목록에서, When 순서 변경을 수행하면, Then 카테고리 노출 순서가 변경된다

- Given 메뉴 목록에, When 표시될 때, Then 메뉴명, 가격, 카테고리, 이미지 썸네일이 포함된다

**Acceptance Criteria**:
- Given 메뉴 관리 화면에서, When 메뉴 추가 버튼을 누르면, Then 메뉴 등록 폼(메뉴명, 가격, 설명, 카테고리, 이미지)이 표시된다
- Given 등록 폼에서, When 이미지 파일을 선택하면, Then 이미지가 로컬 파일 시스템에 업로드된다
- Given 등록 폼에서, When 모든 필수 필드를 입력하고 저장하면, Then 메뉴가 등록되고 전체 매장에 반영된다
- Error: 필수 필드(메뉴명, 가격, 카테고리) 미입력 시 검증 에러 표시
- Error: 가격이 유효 범위를 벗어난 경우 검증 에러 표시

## US-S09-4: 메뉴 커스텀 옵션 그룹 관리
**As a** 슈퍼 관리자, **I want to** 메뉴별 커스텀 옵션 그룹(사이즈, 온도, 샷, 시럽, 우유 종류 등)을 관리할 수 있기를 **so that** 고객이 커스텀 주문을 할 수 있다.

**Acceptance Criteria**:
- Given 메뉴 등록/수정 화면에서, When 옵션 그룹 추가 버튼을 누르면, Then 옵션 그룹 설정 폼(그룹명, 선택 유형, 필수 여부)이 표시된다
- Given 옵션 그룹 설정에서, When 선택 유형을 지정하면, Then 단일 선택 또는 다중 선택으로 설정된다
- Given 옵션 그룹에서, When 개별 옵션을 추가하면, Then 옵션명과 추가 금액을 입력할 수 있다
- Given 옵션 그룹을 저장하면, When 고객 화면에서 해당 메뉴를 선택하면, Then 설정한 옵션 그룹이 표시된다
- Given 기존 옵션 그룹에서, When 수정/삭제하면, Then 변경 사항이 즉시 반영된다
- Error: 필수 옵션 그룹에 옵션이 0개인 경우 검증 에러 표시

## US-S09-5: 메뉴 수정

- Given 수정 폼에서, When 정보를 변경하고 저장하면, Then 메뉴 정보가 업데이트되고 전체 매장에 반영된다

- Given 확인 팝업에서, When 확인을 누르면, Then 메뉴가 삭제되고 전체 매장에서 제거된다

# INVEST Criteria Verification

| 기준 | 충족 여부 | 설명 |
|------|-----------|------|
| Independent | O | 각 스토리가 독립적으로 구현 가능 (의존성은 있으나 독립 테스트 가능) |
| Negotiable | O | 구현 방식은 협의 가능, 핵심 가치만 고정 |
| Valuable | O | 각 스토리가 사용자에게 직접적인 가치 제공 |
| Estimable | O | Task 수준으로 세분화되어 추정 가능 |
| Small | O | 각 스토리가 단일 기능 단위로 작게 분리됨 |
| Testable | O | Given-When-Then 형식의 상세 AC로 테스트 가능 |

**총 스토리 수**: 36개 MVP 활성 (고객 11개, 매장 관리자 9개, 슈퍼 관리자 16개 중 MVP 14개 + MVP 이후 2개)
