## NFR 질문

### Q1: API 응답 시간 목표
MVP 단계에서 일반 API(메뉴 조회, 주문 생성 등)의 목표 응답 시간은 어느 정도로 설정할까요?

- A) 500ms 이내 (일반적인 웹 앱 수준)
- B) 1초 이내 (MVP 충분)
- C) 특별한 목표 없음 (동작만 하면 됨)

[Answer]: B (1초 이내, MVP 충분)

### Q2: 동시 접속자 규모
MVP 테스트 시 예상하는 동시 접속 규모는?

- A) 1개 매장, 테이블 10개 이하
- B) 1개 매장, 테이블 20~30개
- C) 여러 매장 동시 테스트

[Answer]: A (1개 매장, 10개 이하)

### Q3: 에러 처리 수준
API 에러 응답의 상세도를 어느 수준으로 할까요?

- A) 표준 HTTP 상태 코드 + 간단한 메시지 (예: 400 Bad Request + "필수 옵션을 선택해주세요")
- B) 에러 코드 체계 포함 (예: ORDER_001 + 상세 메시지 + 필드별 검증 에러)
- C) 최소한 (HTTP 상태 코드만)

[Answer]: A (HTTP 상태 코드 + 간단한 메시지)

### Q4: 로깅 수준
서버 로깅을 어느 수준으로 설정할까요?

- A) 기본 (Spring Boot 기본 로깅, 에러만 상세)
- B) 중간 (주문 생성/상태 변경 등 주요 비즈니스 이벤트 로깅)
- C) 상세 (모든 API 요청/응답 로깅)

[Answer]: A (Spring Boot 기본 로깅)

### Q5: 데이터 검증 위치
입력 데이터 검증을 어디서 수행할까요?

- A) 프론트엔드 + 백엔드 양쪽 (이중 검증)
- B) 백엔드만 (서버 신뢰)
- C) 프론트엔드 위주 + 백엔드 최소 검증

[Answer]: A (프론트엔드 + 백엔드 이중 검증)

### Q6: H2 파일 모드 설정
H2 데이터베이스 파일 모드 설정 관련:

- A) 앱 재시작 시 데이터 유지 (파일 모드, 개발 중 데이터 보존)
- B) 앱 재시작 시 데이터 초기화 (인메모리, 깨끗한 시작)
- C) 파일 모드 + 시작 시 기본 데이터 자동 삽입 (seed data)

[Answer]: C (파일 모드 + seed data, 개발 편의성)

### Q7: 이미지 업로드 제한
메뉴 이미지 업로드 제한 설정:

- A) 파일 크기 5MB, 형식 JPG/PNG만
- B) 파일 크기 10MB, 형식 제한 없음
- C) 파일 크기 2MB, 형식 JPG/PNG/WebP

[Answer]: A (5MB, JPG/PNG, MVP에 충분)
