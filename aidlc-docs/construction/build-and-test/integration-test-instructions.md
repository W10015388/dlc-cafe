# Integration Test Instructions

## 목적
Backend API와 Frontend가 올바르게 연동되는지 수동 통합 테스트

## 사전 준비
```bash
# 터미널 1: Backend
cd backend && ./gradlew bootRun

# 터미널 2: Frontend
cd frontend && npm run dev
```

## 테스트 시나리오

### Scenario 1: 고객 주문 플로우
1. http://localhost:5173 접속
2. "고객 (태블릿 설정)" 클릭
3. 매장코드: `CAFE01`, 테이블번호: `1`, 비밀번호: `1234` 입력 → 설정 완료
4. 메뉴 화면 표시 확인 (카테고리 탭, 메뉴 카드)
5. "아메리카노(HOT)" 클릭 → 옵션 모달 표시
6. 사이즈 "Grande" 선택 → 장바구니 담기
7. 장바구니 이동 → 수량 확인, 금액 5,000원 확인
8. "주문하기" → 주문 확인 → "주문 확정"
9. 주문 번호 표시 확인
10. 주문 내역 탭 → 주문 상태 "대기중" 확인

### Scenario 2: 관리자 주문 관리
1. 새 탭에서 http://localhost:5173 접속
2. "매장 관리자" → "프리미엄카페 강남점" 선택
3. 대시보드에서 테이블 1 카드 확인 (주문 있음)
4. 테이블 1 클릭 → 주문 상세 모달
5. "다음 상태" 클릭 → 상태 "준비중"으로 변경
6. 고객 탭에서 주문 상태 "준비중" 실시간 반영 확인 (SSE)
7. "다음 상태" 클릭 → "완료"
8. "이용완료" 클릭 → 테이블 리셋

### Scenario 3: 슈퍼 관리자 메뉴 관리
1. http://localhost:5173 접속 → "슈퍼 관리자"
2. 매장 탭 → 매장 목록 확인 (프리미엄카페 강남점)
3. 카테고리 탭 → 6개 기본 카테고리 확인
4. 메뉴 탭 → 메뉴 목록 확인
5. "+ 메뉴 등록" → 새 메뉴 등록 → 목록에 반영 확인
6. 관리자 탭 → 관리자 계정 생성/삭제

## 예상 결과
- 모든 시나리오가 에러 없이 완료
- SSE 실시간 업데이트 동작 확인
- 장바구니 새로고침 후에도 유지 (localStorage)
