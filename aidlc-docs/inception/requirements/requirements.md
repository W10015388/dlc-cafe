# 테이블오더 서비스 - 요구사항 문서 (프리미엄 카페)

## Intent Analysis

- **User Request**: 프리미엄 카페 테이블오더 서비스 구축 (커스텀 주문 옵션 지원, 전국 매장 통합 관리)
- **Request Type**: New Project (신규 프로젝트)
- **Scope Estimate**: Multiple Components (Frontend + Backend + Database)
- **Complexity Estimate**: Moderate (실시간 통신, 세션 관리, 다중 사용자 유형, 커스텀 옵션)

---

## 1. 기술 스택

| 영역 | 기술 |
|------|------|
| Backend | Java + Spring Boot |
| Frontend | React |
| Database | H2 (파일 모드, 추후 PostgreSQL 전환 가능) |
| 실시간 통신 | SSE (Server-Sent Events) |
| 인증 | 테이블 태블릿 토큰 기반 (관리자 인증 없음) |
| 이미지 저장 | 로컬 파일 시스템 |
| 배포 환경 | 로컬 개발 환경 우선 (배포는 추후 결정) |

---

## 2. Functional Requirements

### 2.1 고객용 기능 (Customer Features)

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 태블릿 초기 설정 시 매장 식별자, 테이블 번호, 테이블 비밀번호 입력
- 고객은 별도 로그인 절차 없이 즉시 주문 가능
- 1회 로그인 성공 후 저장된 정보로 자동 로그인
- 로그인 정보 로컬 저장

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면으로 항상 표시
- 카테고리별 메뉴 분류 및 표시
- 기본 카테고리: 커피(HOT), 커피(ICE), 논커피 음료, 스무디/프라페, 티/에이드, 디저트
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (페이지 새로고침 시에도 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 주문 확정
- 주문 성공 시: 주문 번호 표시, 장바구니 자동 비우기, 5초 후 메뉴 화면 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 식별 정보, 테이블 식별 정보, 주문 메뉴 목록(메뉴명, 커스텀 옵션, 수량, 단가), 총 주문 금액, 세션 ID
- 테이블 세션 시작: 해당 테이블의 첫 주문 시 새로운 세션이 자동 생성됨

#### FR-C05: 주문 내역 조회
- 주문 시간 순 정렬
- 주문별 상세: 주문 번호, 주문 시각, 메뉴 및 수량, 커스텀 옵션, 금액, 상태(대기중/준비중/완료)
- 주문 상태 SSE 기반 실시간 업데이트
- 현재 테이블 세션 주문만 표시 (이전 세션 제외)

### 2.2 관리자용 기능 (Admin Features)

#### FR-A01: 관리자 진입
- 로그인 없이 역할 선택으로 바로 진입
- 슈퍼 관리자 / 매장 관리자 중 선택
- 매장 관리자 선택 시 매장 선택 화면 표시

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 주문 목록 실시간 업데이트 (2초 이내 표시)
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드에 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중/준비중/완료)
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 테이블별 필터링

#### FR-A03: 테이블 관리
- 테이블 태블릿 초기 설정 (테이블 번호, 비밀번호 설정, 16시간 세션 생성)
- 주문 삭제 (확인 팝업, 즉시 삭제, 총 주문액 재계산)
- 테이블 세션 처리 (이용 완료 시 주문 내역을 과거 이력으로 이동, 테이블 리셋)
- 과거 주문 내역 조회 (시간 역순, 날짜 필터링)

#### FR-A04: 메뉴 관리 (MVP 포함)
- 메뉴 CRUD (등록/조회/수정/삭제)
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지
- 이미지 파일 업로드 (로컬 파일 시스템 저장)
- 메뉴 노출 순서 조정
- 필수 필드 검증, 가격 범위 검증
- 메뉴별 커스텀 옵션 그룹 관리 (옵션 그룹 생성/수정/삭제)
- 메뉴는 전체 체인 공통 (슈퍼 관리자가 관리)

#### FR-A04-1: 카테고리 관리
- 카테고리 CRUD (등록/조회/수정/삭제)
- 기본 카테고리: 커피(HOT), 커피(ICE), 논커피 음료, 스무디/프라페, 티/에이드, 디저트
- 카테고리 노출 순서 조정
- 카테고리는 전체 체인 공통 (슈퍼 관리자가 관리)

#### FR-C06: 커스텀 주문 옵션
- 메뉴별 커스텀 옵션 선택 (예: 사이즈, 온도, 샷 추가, 시럽, 우유 종류 등)
- 옵션 그룹별 단일 선택 또는 다중 선택 지원
- 옵션별 추가 금액 반영
- 필수 옵션과 선택 옵션 구분
- 선택한 옵션이 장바구니 및 주문 내역에 표시

### 2.3 관리자 계정 구조

#### FR-A05: 슈퍼 관리자 + 매장 관리자 계층 구조
- 슈퍼 관리자: 매장 생성/관리, 매장 관리자 계정 생성, 전체 체인 공통 메뉴/카테고리 관리
- 매장 관리자: 자신의 매장 내 테이블/주문 관리
- MVP 범위: 매장 CRUD, 매장 관리자 계정 생성, 메뉴/카테고리 관리 (전국 모니터링/시스템 설정은 MVP 이후)

## 3. Non-Functional Requirements

### NFR-01: 성능
- MVP 단계: 단일 매장 테스트 수준
- SSE 기반 실시간 주문 표시 2초 이내

### NFR-02: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적인 카드 형태 레이아웃
- 명확한 시각적 계층 구조

### NFR-03: 데이터 관리
- 장바구니: 클라이언트 측 로컬 저장
- 주문 이력: 데이터베이스 저장 (OrderHistory)
- 세션 ID로 주문 그룹화

---

## 4. 제외 기능 (Out of Scope)

- 결제 처리 (카드, 현금, PG사 연동)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA)
- 이미지 리사이징/최적화
- 푸시/SMS/이메일 알림
- 주방 기능 (주방 전달, 재고 관리)
- 데이터 분석/매출 리포트
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어)
- 예약/리뷰/다국어 기능
