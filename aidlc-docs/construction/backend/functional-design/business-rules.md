# Business Rules - Backend

## BR-01: 주문 상태 전이
- 허용: PENDING → PREPARING → COMPLETED (순방향만)
- 금지: 역방향 전이 (COMPLETED → PREPARING 등)
- 검증: 현재 상태에서 다음 상태로만 전이 가능

## BR-02: 테이블 세션 라이프사이클
- 세션 시작: 해당 테이블의 첫 주문 생성 시 자동으로 새 세션 생성
- 세션 종료: 관리자가 이용 완료 처리 시에만 종료 (자동 만료 없음)
- 활성 세션: 테이블당 최대 1개의 활성 세션만 존재
- 세션 종료 시: 모든 주문을 OrderHistory로 아카이빙 → 세션 비활성화

## BR-03: 주문 생성 규칙
- 주문 항목 최소 1개 이상 필수
- 각 항목의 필수 옵션 그룹은 반드시 선택되어야 함
- 단일 선택 옵션 그룹: 정확히 1개 선택
- 다중 선택 옵션 그룹: 0개 이상 선택
- 주문 금액 = SUM(각 항목의 (메뉴 기본가 + 선택 옵션 추가금) × 수량)
- 주문 번호: 자동 생성 (매장코드-날짜-순번, 예: CAFE01-20260317-001)

## BR-04: 메뉴 검증
- 메뉴명: 필수, 최대 100자
- 가격: 필수, 0원 이상
- 카테고리: 필수, 존재하는 카테고리만 허용
- 이미지: 선택, 파일 업로드 시 로컬 저장

## BR-05: 옵션 그룹 검증
- 필수 옵션 그룹: 최소 1개 이상의 옵션 필요
- 옵션 추가 금액: 0원 이상

## BR-06: 매장 검증
- 매장명: 필수, 최대 100자
- 매장 식별자(storeCode): 필수, 유니크, 최대 50자

## BR-07: 테이블 검증
- 테이블 번호: 필수, 매장 내 유니크
- 비밀번호: 필수
- 태블릿 인증: storeCode + tableNumber + password 일치 시 인증 성공

## BR-08: 주문 삭제 규칙
- 관리자만 삭제 가능
- 삭제 시 테이블 총 주문액 재계산
- 삭제된 주문은 SSE 이벤트로 고객 화면에 반영

## BR-09: 이용 완료 규칙
- 활성 세션이 있는 테이블만 이용 완료 가능
- 이용 완료 시: 모든 주문(상태 무관) → OrderHistory로 JSON 스냅샷 저장
- 이용 완료 후: 테이블 세션 비활성화, 새 고객은 첫 주문 시 새 세션 시작

## BR-10: 카테고리 삭제 규칙
- 해당 카테고리에 메뉴가 있으면 삭제 불가 (먼저 메뉴 이동/삭제 필요)

## BR-11: 매장 삭제 규칙
- 활성 주문이 있는 매장은 경고 표시 후 삭제 가능 (cascade)
